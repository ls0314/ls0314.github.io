    Embedding+位置编码

    Inputs输入为数据对应的Token内容。#将词Token化意义是减少词表的大小。

    Input Embedding将Token映射为相应的词向量后传递给Positional Encoding进行位置编码，将位置信息嵌入词向量。

    Encoder和Decoder

    encode编码器其作用是将输入的向量转换为语义上的向量，而decoder解码器是要将语义上的向量转义为符合其语义的结果形式对应的向量，最后此向量通过Liner和softmax层变成自然语言形式。

    #从NLP角度来说encoder就是负责将一段话从不同的关键词角度聚合成对应的词向量（即每一个向量都是代表这个段落，只是侧重点不同）并传入decoder，decoder负责将自己上一轮输出的内容对应的词向量与encoder传入的向量进行汇总并进行本轮的输出。

    我们从Attention Is All You Need中介绍的模型结构来看，encoder部分大致由两个模块组成，feed forward和Multi-Head模块。decoder部分大致由三个模块组成feed forward、Multi-Head和Masked Multi-Head模块。其中二者的feed forward为前馈神经网络，encoder中的Multi-Head负责将带有位置信息的词向量进行注意力处理得到各词向量间的注意力权重关系，decoder中的Multi-Head负责将自己经过Masked Multi-Head处理的上一轮的输出结果的向量与encoder输出的向量汇总在一起以保证上下文和生成结果的联系。Masked Multi-Head是带掩盖的Multi-Head模块，通过特殊处理来遮盖掉时序对注意力权重的影响。

    #encode的工作过程是并行的一次性的而decode是自回归的。

    Multi-Head Attention 多头注意力机制

    注意力机制公式

    在注意力机制中其中Q代表的是含义是查询，即注意力计算的主体。K和V是相联系的键值对，K的形式与Q相同，其公式的物理含义是Q通过K与V的内在联系计算得出对应的相关度，从而得出与Q对应的键值对V*。

    #个人理解K和V是一些具有某种映射关系的数据信息，Q是一条查询，要计算Q和K的每条信息的相似度再通过这 个相似度来预测Q对应的V*的值。

    上面阐述的是注意力机制，而当变成自注意力机制时，会使QKV三者都变成一个段落本身即QKV三者相同。

    自注意力机制公式

    由于在自注意力机制计算的过程中只有简单的数据计算，为引入神经网络以达到弱化Embedding和调节误差的作用，Self-Attention使用W_q，W_k，W_v三个神经网络对段落本身X进行处理得到简单变换后的QKV。

    #自注意力机制注重的是信息的聚合，即对Q矩阵中每个向量按照与其他向量与其相关度进行聚合。举例来说就是对同一段话（Q）按照不同的侧重点（Q矩阵中向量）进行信息间的联系。

    #由于位置编码的存在，相邻的向量间的相关性会有略微提升。

    上述图片同样来自Attention Is All You Need中将Multi-Head Attention模块展开，可以看出其是对注意力公式的可视化展开，query矩阵与key矩阵进行点乘后可以得到每个单词与其他单词的关注度分数。将得到的结果进行Scale处理并进行softmax计算得到注意力权重矩阵。将得到的注意力权重矩阵与value矩阵点乘可以得到最终的Q对应注意力系数矩阵。

    #Scale处理即，此操作可以很好的防止乘法带来的梯度爆炸问题。可以保证方差不变。

    #softmax计算后，较高的分数会得到增强，较低的分数会被抑制。

    对于Multi-Head而言，其含义是在上述操作完成后通过concat将多个head通过特定的权重组合到一起。仍用开篇的例子对Transformer的多头机制具体运行机制进行绘制。

    #与正常的自注意力机制有所不同的是，在Transformer所使用的Multi-Head Attention中，将每一个N维的向量 分成x个M维的向量（保证x*M=N），对每个M维的向量使用单独的成组的权重矩阵进行Self-Attention计算， 最后将所有计算结果首尾相连的通过W^o矩阵融合到一起。根据Attention is All You Need所说，使用单个 Self-Attention会削弱向量中的信息，采取多头机制可以很好的解决这个问题，且二者具有相似的计算量。

    Multi-Head计算公式

    Masked Multi-Head的主要作用是防止当前的Token“偷看”未来的Token。即当前Token只与它序列之前的Token进行信息的聚合。

    Add&Norm

    此结构代表的是残差连接和Layer Normalization

    #残差连接简单说就是将模块处理前的数据和处理后的数据进行合并处理，避免梯度消失的出现，可以很好的保证 网络的深度

    #Layer Normalization层归一化，与之对比的是Batch Normalization批归一化，个人理解二者区别为横向和纵向的区别，以“明天会更好”，“欢迎新同学”为例，BN可能会更在意“明”和“欢”的联系，而忽略了“明天会更好”，“欢迎新同学”作为一句话本身的内涵。而LN可以很好的解决这个问题，会更加注意“明天会更好”本身之间的联系。

    ed forward 前馈神经网络

    在Transformer中的feed forward就是简单的前馈神经网络，假设Multi-Head模块传入的是N维向量，则此前馈神经网络结构为N维的输入层，4N+1维隐藏层，N维输出层组成，输入层到隐藏层（即N到4N+1）的目的是放大特征，而隐藏层到输出层的目的是特征提取或特征抽取。

    Decoder-only架构

    Decoder-only是Transformer模型的变种之一，其丢弃了传统Transformer的Encoder部分，只保留Decoder部分。此架构为GPT，Llama等大语言模型的核心架构，在放弃了Encoder-Decoder的强耦合设计下仍保留对生成任务的适配性，在Decoder-only中不在区分输入和输出的特性更适用于生成任务。

    Transformer复杂度计算

    在Transformer模型中主要的复杂度来在Multi-Head和Feed forward两个部分。假设采用和参数量计算部分一样的变量，且另设Encoder和Decoder的输入序列长度分别为

    在Multi-Head中涉及计算的有三个部分，分别是线性变换得到Q，K，V矩阵，其计算量为$ n_{e_{or}d}3d^2 此部分的复杂度为 O（n_{e_{or}d}d^2) ; 各向量间的相关性计算，其计算量为 n_{e_{or}d}^2d 此部分的复杂度为 O（n_{e_{or}d}^2d） ; 相关性矩阵和V进行加权求和，其计算量为 n_{e_{or}d}^2d 此部分的复杂度为 O（n_{e_{or}d}^2d） 。所以Multi − Head部分的复杂度就为 O（n_{e_{or}d}^2d） $。

    对于Feed forward的主要计算量是两个全连接层，计算量为$ 2n_{e_{or}d}d*4d 则此部分复杂度为 O(n_{e_{or}d}d^2) $

    所以一层Encoder的复杂度为$ O(n_ed^2 + n_e^2d) , N层就是 O(N(n_ed^2 + n_e^2d)) $。

    一层Decoder的复杂度为$ O(n_dd^2 + n_d^2d + n_en_dd) , N层就是 O(N(n_dd^2 + n_d^2d + n_en_dd)) $。

    #在Decoder中和Encoder连接的Attention部分的复杂度计算为$ O(n_en_dd) $。

    Transformer在CV领域的变种模型

    Vision Transformer模型

    Vision Transformer(ViT)模型在2020年由谷歌团队在《AN IMAGE IS WORTH 16X16 WORDS: TRANSFORMERS FOR IMAGE RECOGNITION AT SCALE》一文中提出，其模型架构图如下图所示。

    与传统Transformer应用于NLP领域不同ViT是应用于CV领域，ViT构建了一种只依靠Transformer来解决图像分类任务的结构，其主要有几个关键点。

    • Patch Embedding ：为避免直接将图片拉伸成序列出现的平方爆炸，ViT采取了将图片分成多个16X16的Patch的方法，将分好的每一个Patch都视作一个类似word的Token，即从原先的将像素点视作Token转变为将Patch视作Token。处理好的Patch要传入图中的Linear Projection of Flattened Patches（Embedding）层，作用是将Patch展成序列，其维度是16X16X3 = 768维。此时的形式就可以很好的用Transformer来进行处理了。
    • class Embedding：借鉴了BERT的Class Token设计，将其作为整个Transformer Encoder的输出，来进行最后的分类任务。值得一提的是，在ViT这篇论文中它自己提到了即使不使用这个cls这个设计也可以通过采用GAP并调节合适的学习率来达到相同的效果。
    • Position Embedding：使用的标准的1D Position Embedding（在ViT文中也提到即使用2D Position Embedding最终也只能得到和1D差不多的效果）。

    ViT的主旨就要在避免对Transformer进行改变的基础上构建模型，所以它和Transformer的不同之处主要在预处理部分和最后的分类输出部分.

    #因为ViT只是实现图像分类，不需要执行生成任务所以只使用了Transformer的Encoder部分进行图像的语义理解。
